## 1. 빠른 시작

**`serve.bat`를 더블클릭**하면 끝입니다.

- Python이 이미 설치되어 있으면 → 즉시 서버 시작 + 브라우저 열림
- Python이 없으면 → **자동으로 Python 3.13을 설치**한 후 서버를 시작합니다

서버는 **백그라운드에서 실행**되므로 콘솔 창을 계속 띄워둘 필요가 없습니다.
사용이 끝나면 콘솔 창에서 **아무 키를 누르면** 서버가 종료됩니다.

| 동작 | 방법 |
|------|------|
| **시작** | `serve.bat` 더블클릭 → 브라우저 자동 열림 |
| **종료** | 콘솔 창에서 아무 키 누르기 또는 창 닫기 |
| **재시작** | 종료 후 `serve.bat` 다시 더블클릭 |

## 2. Python 자동 설치

Python이 설치되어 있지 않은 경우 `serve.bat`가 자동으로 처리합니다:

1. Python 3.13 설치 파일을 다운로드합니다 (인터넷 연결 필요)
2. 현재 사용자 계정에 자동 설치합니다 (관리자 권한 불필요)
3. PATH 환경변수에 자동 등록합니다
4. 설치 완료 후 서버를 시작합니다

> **참고**: Python이 이미 설치되어 있으면 이 과정은 자동으로 건너뜁니다. 중복 설치되지 않습니다.

### 자동 설치가 실패하는 경우

인터넷 연결 문제 등으로 자동 설치가 실패하면 수동으로 설치하세요:

1. **[https://www.python.org/downloads/](https://www.python.org/downloads/)** 에 접속합니다.
2. **"Download Python 3.x.x"** 버튼을 클릭합니다.
3. 설치 파일을 실행하고 **☑ Add Python to PATH** 를 **반드시 체크**합니다.
4. **"Install Now"** 를 클릭합니다.
5. 설치 완료 후 `serve.bat`를 다시 실행합니다.

## 4. Files — 파일 관리

S3 스토리지의 게임 데이터 파일을 관리하는 탭입니다.

### 4-1. 화면 구성

```
┌─────────────────────────────────────────────┐
│  Connected ●  │  1,234 files  │  Upload  │  New Folder  │
├──────────┬──────────────────────────────────┤
│  폴더 트리  │  📁 현재 폴더 경로 (빵 부스러기)        │
│  (좌측)    │  ┌─────────────────────────────┤
│           │  │ 📁 폴더명     📄 파일명  ⬇   │
│           │  │ 📁 ...       📄 ...     ⬇   │
│           │  │ ...                         │
└──────────┴──────────────────────────────────┘
```

- **좌측 패널**: 전체 폴더 트리 (확장/축소 가능)
- **우측 패널**: 현재 폴더의 파일 목록
- **상단 바**: 연결 상태 + 파일 수 + 업로드/폴더 생성 버튼

### 4-2. 폴더 탐색

| 동작 | 좌측 트리 | 우측 파일 목록 |
|------|----------|---------------|
| **한 번 클릭** | 하위 폴더 펼침/접기 | 해당 폴더로 이동 |
| **더블 클릭** | 해당 폴더로 이동 | — |

- 상단 **경로 바(빵 부스러기)**의 각 구간을 클릭하면 상위 폴더로 이동합니다.

### 4-3. 파일 업로드

1. 원하는 폴더로 먼저 이동합니다.
2. 상단의 **"Upload Files"** 버튼 클릭 또는 파일을 **드래그 앤 드롭**합니다.
3. 업로드 진행률이 표시되며, 완료 시 파일 목록이 자동 갱신됩니다.

> **주의**: 업로드는 **현재 보고 있는 폴더**에 저장됩니다. 경로를 반드시 확인하세요.

### 4-4. 파일 다운로드

- 파일 행의 **⬇ 버튼**을 클릭합니다.
- 브라우저 기본 다운로드 위치에 저장됩니다.

### 4-5. 새 폴더 만들기

1. **"New Folder"** 버튼 클릭
2. 이름 입력 → **"Create"** 클릭

### 4-6. 연결 관리

- **Connected** 버튼에 마우스를 올리면 → **"Disconnect"**로 전환
- 클릭하면 연결 해제 → 다시 **"Connect"**로 재연결

## 5. Dashboard — 시스템 모니터링

QA Supporter 봇의 전체 시스템 상태를 실시간으로 확인하는 탭입니다.

### 5-1. 대시보드 카드 구성

| 카드 | 내용 | 갱신 주기 |
|------|------|----------|
| **System Health** | 봇 프로세스 상태, 스케줄 수, 최근 동기화 기록 | 페이지 로드 시 |
| **Cache Status** | Wiki/GDI/Jira 노드 수, body 적재율, DB 크기 | 페이지 로드 시 |
| **Query Performance** | 쿼리 수, 캐시 히트율, 평균 응답 시간, 소스별 분포 | 페이지 로드 시 |
| **Active Schedules** | 활성 스케줄 목록, 각 스케줄의 구성 정보 | 페이지 로드 시 |
| **Recent Claims** | 최근 접수된 클레임 10건 (날짜, 카테고리, 사용자, 내용) | 페이지 로드 시 |
| **Activity Log** | 통합 이벤트 로그 (쿼리/클레임/동기화) 최근 30건 | 페이지 로드 시 |

### 5-2. System Health

- **Bot Status**: 봇 프로세스가 실행 중인지 표시 (🟢 Running / 🔴 Stopped)
- **Schedules**: 등록된 스케줄 알림 개수
- **Last Sync**: 가장 최근 캐시 동기화 결과 (소스, 시간, 소요시간)

### 5-3. Cache Status

- **소스별 노드 수**: Wiki / GDI / Jira 각각의 캐시된 문서 수
- **Body 적재율**: 본문이 실제로 캐시에 저장된 비율 (%)
- **색상 바**: 진행률을 시각적으로 표시
  - 🔵 Wiki / 🟣 GDI / 🩵 Jira

### 5-4. Query Performance

- **Total Queries**: 전체 쿼리 수
- **Cache Hit Rate**: 캐시에서 응답한 비율 (높을수록 좋음)
- **Avg Response Time**: 평균 응답 시간
- **소스별 바 그래프**: Wiki/GDI/Jira 쿼리 분포

### 5-5. Active Schedules

- config.json에 등록된 스케줄 알림 목록
- 채널별, 시간대별 필터 지원

### 5-6. Recent Claims

- 최근 클레임 10건을 카드 형태로 표시
- 각 카드: 날짜 / 카테고리 태그 / 사용자 / 내용 요약
- 카테고리별 필터 지원

### 5-7. Activity Log

- 쿼리, 클레임, 동기화 이벤트를 시간순으로 통합 표시
- 컬럼: 시간 / 소스 / 이벤트 타입 / 상세 정보
- 이벤트별 필터 지원

## 6. 문제 해결

| 증상 | 원인 | 해결 |
|------|------|------|
| 자동 설치 실패 | 인터넷 연결 불가 | 수동 설치 (위 가이드 참조) |
| "Python을 찾을 수 없습니다" | PATH 미등록 | PC 재시작 후 다시 실행 |
| "Connect" 후 "Failed" | 서버 미실행 | serve.bat를 다시 실행 |
| 브라우저가 안 열림 | 기본 브라우저 설정 | `http://localhost:9090/s3_manager.html` 직접 입력 |
| 업로드 성공 후 파일 안 보임 | 잘못된 폴더에서 업로드 | 상단 경로 확인 후 올바른 폴더에서 업로드 |
| Dashboard 데이터가 안 뜸 | 봇 미실행 또는 캐시 DB 없음 | 봇 실행 상태 확인 |
| 페이지 로딩이 느림 | 네트워크 환경 | VPN 연결 상태 확인 |

### 포트 충돌 시

다른 프로그램이 9090 포트를 사용 중이면:

1. 명령 프롬프트(cmd)를 엽니다.
2. 해당 폴더로 이동: `cd "배포폴더경로"`
3. 다른 포트로 실행: `python s3_server.py --port 8080`
4. 브라우저에서 `http://localhost:8080/s3_manager.html` 접속
